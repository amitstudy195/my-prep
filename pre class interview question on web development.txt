1.)what is web development?
Ans- Web development is the process of building, programming, and maintaining websites and web applications that run on the internet.

2.)what is the difference b/w a website and a web application?
Ans- The fundamental difference is that a website is meant to be read or viewed, while a web application is designed for user interaction and performing tasks. While both are accessed through a web browser and require internet access, web apps behave more like software programs.

3.)what is a client and what is server?
Ans- A client is an application or device that requests services or resources, like a web browser. A server is a system or software that manages, stores, and provides those requested services, such as a website host.

4.)what is the role of the browser in web communication?
Ans- The browser acts as the client-side interface in web communication. It translates user actions into requests, sends them over the internet using standardized protocols, and renders returned code (HTML, CSS, JavaScript) into the visual, interactive web pages you see.

5.)what is the internet?
Ans- The internet is a massive, worldwide network of computers, phones, and other devices all connected to each other. 

6.)what is DNS and why it is needed?
Ans- DNS (Domain Name System) is the internet's equivalent of a phonebook. It automatically translates human-readable website names (like google.com) into the numerical IP addresses (like 192.168.1.1) that computers use to locate and connect to each other.

7.)what happend when we type a URL in the browser?
Ans- When you type a URL into a browser and press Enter, the browser translates the text into an actual webpage in a matter of milliseconds. It does this by locating the website, establishing a secure connection, retrieving the requested files, and assembling them visually on your screen.

8.)what is the IP address?
Ans- An IP (Internet Protocol) address is a unique numerical label assigned to every device connected to a computer network. It acts like a digital postal address, allowing devices (such as computers, phones, and servers) to identify each other and route data accurately across the internet or a local network.

9.)difference between HTTP and HTTPS?
Ans- The fundamental difference between HTTP and HTTPS is security. HTTPS encrypts data so it cannot be intercepted by hackers, while HTTP sends data as unencrypted plain text.

10.)what is the domain name?
Ans- A domain name is the human-readable web address used to find a website on the internet (e.g., google.com).

11.)what is the static website?
Ans- A static website serves pre-built HTML, CSS, and JavaScript files exactly as they are stored on a server. Every visitor receives the exact same content, and the pages do not interact with databases or server-side scripts.

12.)what is the dynamic website?
Ans- A dynamic website is a site that adapts its content, appearance, or features in real-time based on the user's behavior, preferences, or the time of day. Unlike static sites, which show the exact same information to everyone, dynamic sites pull information from databases to build pages on the fly.

13.)give examples of static and dynamic websites?
Ans- Static websites deliver the exact same pre-built files to every visitor, while dynamic websites generate unique content in real time based on user behavior, databases, or logins.
Examples of Static Websites- Company Brochure Sites,Restaurant Menu Pages,Personal Portfolios and Résumés,Marketing Landing Pages,Documentation Portals
Examples of Dynamic Websites- E-Commerce Marketplaces,Social Media Networks,Streaming Services,News Portals,Web Search Engines

14.)which technolgies are used to build dynamic websites?
Ans- Dynamic websites change content based on user interactions. They are built using a combination of front-end technologies for browser interaction, back-end technologies for server processing, and databases to store and retrieve data.

15.)what is the full stack development?
Ans- Full stack development is the process of building both the front-end (user-facing interface) and back-end (server, logic, and database) of a web application. A full-stack developer handles the entire end-to-end development cycle, from crafting user interactions to managing server infrastructure and data storage.
